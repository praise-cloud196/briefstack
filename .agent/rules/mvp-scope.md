# MVP SCOPE — BRIEFSTACK

## PURPOSE
This file defines exactly what is in scope for MVP and prevents feature creep.

---

## 1. MVP GOAL

Users must be able to:

- input a content idea
- receive a structured strategic brief
- edit it
- save it
- export it

That is the entire product.

---

## 2. INCLUDED IN MVP

### Core Features

- User authentication
- AI brief generation
- Brief editor
- Save brief functionality
- Brief history page
- Export brief (PDF / Markdown / Copy)

---

## 3. EXCLUDED FROM MVP (STRICT)

DO NOT BUILD:

- AI writing or blog generation
- Keyword research tools
- SEO analytics dashboards
- Competitor analysis
- SERP scraping
- Content calendars
- Team collaboration
- Multi-user permissions
- Workspaces
- Notifications
- Scheduling tools
- Marketing automation
- AI chat assistant interface

---

## 4. MVP SUCCESS DEFINITION

MVP is successful if:

- users generate useful briefs in under 60 seconds
- briefs are actually usable by writers
- users export briefs regularly
- users return to create multiple briefs

---

## 5. PRODUCT FOCUS RULE

If a feature does not improve:

- speed of brief creation
- clarity of strategy
- usefulness of output

IT DOES NOT BELONG IN MVP.

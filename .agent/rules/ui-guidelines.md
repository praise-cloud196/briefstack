# UI GUIDELINES — BRIEFSTACK

## Purpose
Define the visual and interaction system for BriefStack to ensure a clean, focused, professional SaaS experience.

---

## 1. DESIGN PHILOSOPHY

BriefStack UI should feel:

- minimal
- fast
- calm
- professional
- distraction-free

Avoid:
- dashboards with too many panels
- flashy animations
- unnecessary gradients
- cluttered layouts

---

## 2. PRIMARY LAYOUT

The core layout is a split workspace:

LEFT PANEL:
- Input form (topic, audience, content type, funnel stage)

RIGHT PANEL:
- Generated AI brief

This is the default and most important screen.

---

## 3. TYPOGRAPHY

- Use clean sans-serif fonts
- Strong hierarchy between headings and body text
- Generated briefs should feel like documents, not chat messages

---

## 4. COMPONENT STYLE

Buttons:
- simple
- solid or light outline
- no heavy shadows

Cards:
- subtle borders
- generous spacing
- minimal decoration

Forms:
- clear labels
- simple inputs
- no unnecessary validation noise

---

## 5. INTERACTION DESIGN

- Prioritize speed over animation
- Instant feedback on actions
- No blocking modals unless necessary
- Editing should feel inline and natural

---

## 6. GENERATED BRIEF DISPLAY

Generated briefs should:

- look like a structured document
- be easy to scan
- use clear section headers
- support inline editing

---

## 7. EMPTY STATES

Empty states should:

- guide users to create their first brief
- avoid over-explaining
- stay minimal and action-focused

---

## 8. RESPONSIVENESS

- Desktop-first experience
- Must degrade cleanly on tablet
- Mobile is secondary (view-only acceptable)

---

## 9. FINAL UX PRINCIPLE

Every screen must answer:

“Does this help the user create a better content brief faster?”

If not, remove it.
